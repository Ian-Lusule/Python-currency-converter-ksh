# Contributing to python-currency-converter-ksh

We welcome contributions to `python-currency-converter-ksh`!  Whether you're a seasoned developer or just starting out, your help is valuable. This document outlines how to contribute to the project.

## Getting Started

1. **Fork the repository:** Create your own fork of the `python-currency-converter-ksh` repository on GitHub.

2. **Clone your fork:** Clone your forked repository to your local machine:

   ```bash
   git clone <your-fork-url>
   ```

3. **Create a branch:** Create a new branch for your feature or bug fix:

   ```bash
   git checkout -b <your-branch-name>
   ```

4. **Install dependencies:**  Make sure you have the required dependencies installed.  You can find these in the `requirements.txt` file. Use a virtual environment for best practice:

   ```bash
   python3 -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```

5. **Make your changes:** Implement your feature or fix the bug.  Follow the coding style and conventions used in the existing codebase.

6. **Test your changes:** Thoroughly test your changes to ensure they work correctly and don't introduce any regressions.  We recommend writing unit tests for any new functionality.

7. **Commit your changes:** Commit your changes with clear and concise commit messages:

   ```bash
   git add .
   git commit -m "Your descriptive commit message"
   ```

8. **Push your branch:** Push your branch to your forked repository:

   ```bash
   git push origin main
   ```

9. **Create a pull request:** Create a pull request on GitHub to merge your changes into the main repository.  Provide a clear description of your changes and address any feedback you receive.


## Coding Style

We follow PEP 8 style guidelines.  Use a consistent style throughout your code.  You can use tools like `pylint` or `flake8` to check your code for style issues.


## Testing

We use unit testing to ensure the quality of the code.  All new features should be accompanied by comprehensive unit tests.  We use the `unittest` framework.


## Documentation

All new features and significant changes should be documented clearly.  Update the relevant sections of the documentation as needed.  Use docstrings to document your code.


## Issues

If you encounter any issues or have suggestions for improvement, please open an issue on the GitHub repository.


## Code of Conduct

We follow a code of conduct based on respect, inclusivity, and collaboration.  Please be mindful of this when interacting with the community.

We appreciate your contributions!  Thank you for helping to make `python-currency-converter-ksh` a better project.
